# ADR-003: Estado Global con Zustand

## Contexto
App offline con estado que debe persistir entre sesiones. Se consideraron Redux Toolkit, Jotai, y Context API.

## Decisión
**Zustand v5 + middleware `persist` + AsyncStorage**.

### Alternativas descartadas
| Alternativa | Motivo de rechazo |
|-------------|-------------------|
| **Redux Toolkit** | Boilerplate excesivo para ~12 campos de estado. Slice, reducer, action, dispatch. |
| **Jotai** | Similar a Zustand pero con menos ecosistema de persistencia. |
| **Context API** | Re-renderizados innecesarios (sin selectores). Requiere Context + Reducer manual. |

### Por qué Zustand
1. **Selectores**: cada screen solo escucha los campos que necesita → sin re-renderizados extras
2. **Persistencia**: middleware `persist` → 1 línea de código, serializa automáticamente a JSON
3. **Mutabilidad inmutable**: usa `set()` con spread operator (similar a React setState)
4. **TypeScript**: tipado completo sin configuración extra

### Patrón de uso
```typescript
// Selector parcial (solo escucha lo que necesita)
const theme = useAppStore((state) => state.theme);

// Acción
const toggleTheme = useAppStore((state) => state.toggleTheme);
```

## Consecuencias
- El store completo se guarda bajo la clave `himnario-storage` en AsyncStorage
- Los campos nuevos se agregan con merge automático (Zustand no requiere migraciones)
- Backup/Restore: se exporta/importa el JSON completo del store
