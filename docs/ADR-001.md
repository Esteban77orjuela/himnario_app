# ADR-001: Estructura y Decisiones Arquitectónicas

## Contexto
App móvil para músicos de iglesia. 100% offline. Sin backend.

## Decisiones

### 1. Monolito Modular por Capas
**Decisión:** Mantener React Native + Expo sin separar backend/frontend.
**Motivo:** No hay servidor. Toda la lógica corre en el dispositivo.

### 2. Estado Global con Zustand + AsyncStorage
**Decisión:** Usar Zustand con middleware persist en lugar de Redux o Context.
**Motivo:** Menos boilerplate, persistencia automática, tipado nativo con TypeScript.

### 3. NativeWind v2 (no v4)
**Decisión:** Usar NativeWind v2 con plugin de Babel en lugar de v4 con Metro.
**Motivo:** v2 es compatible con Windows y no requiere configuración ESM.

### 4. Scraper con fetch + User-Agent Spoofing
**Decisión:** Usar fetch nativo con cabeceras HTTP personalizadas.
**Motivo:** Evitar dependencias externas pesadas (Puppeteer, Cheerio).

### 5. Transposición con Aritmética Modular Base-12
**Decisión:** Array de 12 notas + operador módulo para transponer.
**Motivo:** O(1) por acorde, sin dependencias externas.

## Consecuencias
- La app no puede sincronizar datos entre dispositivos (deliberado).
- El scraping depende de que LaCuerda no cambie su estructura HTML.
